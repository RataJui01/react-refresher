## Architecture

This is a **UI shell** for an e-commerce catalog — routing and layout are complete, all data is static mock data from `src/data/products.js`. Filter/sort logic, pagination slicing, and global cart state are intentionally left unimplemented (see `spec.md` § 6 for the full list of TODOs).

**Stack:** React 19, Vite 8, Tailwind CSS 4, React Router v7 (declarative mode), shadcn/ui, lucide-react. JavaScript only — no TypeScript.

**React Compiler** is enabled via `babel-plugin-react-compiler`. Avoid manual `useMemo`/`useCallback` optimizations — the compiler handles them.

**Path alias:** `@/` → `src/` (set up by shadcn init, used in every import).

### Route structure (`App.jsx`)

All routes are nested under `RootLayout` (Header + `<Outlet/>` + Footer):

| Path | Page component |
|---|---|
| `/` | `HomePage` |
| `/category/:categorySlug` | `CategoryPage` |
| `/product/:productId` | `ProductDetailPage` |
| `/search?q=` | `SearchResultsPage` |
| `/cart` | `CartPage` |
| `*` | `NotFoundPage` |

### Data layer (`src/data/products.js`)

Synchronous helper functions (`getProductById`, `getCategoryBySlug`, `getProductsByCategory`) are the single source of truth. Pages call these directly — no fetching. To migrate to a real API, replace these functions with React Router loaders or TanStack Query.

### What's wired vs TODO

**Working:** navigation, `useParams` lookup, `useSearchParams` read/write (filter values sync to URL), naive search match, `CartPage` local quantity state.

**Not implemented:** filter/sort actually filtering products, pagination slicing, global cart state (Header badge is hardcoded), Add to Cart / Wishlist actions.

## Tailwind CSS v4

Config lives in `src/index.css` via `@theme inline { ... }` — there is no `tailwind.config.js`. Custom design tokens (terracotta color scheme, Fraunces + Work Sans fonts) are declared there.

## shadcn/ui

Components live in `src/components/ui/` and are generated by the shadcn CLI — they are owned by this project, not an npm package. If a component is missing, run:

```bash
npx shadcn@latest add <component-name>
```
